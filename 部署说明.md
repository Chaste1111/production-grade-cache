## 三、源码分析（输出文档）

### 3.1 事件驱动 + 非阻塞 I/O

**文件**: `配置说明/nginx-event-driven.md`

核心内容：
- `ngx_event_t` / `ngx_connection_t` 核心数据结构
- `ngx_process_events_and_timers()` — worker 进程主循环
- epoll 模块实现（`ngx_epoll_process_events` 等）
- 非阻塞 I/O 全链路（accept → read → write）
- 红黑树定时器系统
- 完整的 HTTP 请求事件流转过程

关键设计：
- 一个线程 + 一个 epoll 实例，避免锁竞争和上下文切换
- ET 边缘触发 + 贪婪读写，最小化系统调用
- 回调驱动，事件层和协议层完全解耦

### 3.2 Master-Worker 进程模型

**文件**: `配置说明/nginx-master-worker.md`

核心内容：
- 为什么选多进程而非多线程
- Master 管进程、Worker 管请求的职责分离
- 信号驱动的进程管理（reload / reopen / upgrade）
- 热重载：新旧 worker 交替，零中断
- 故障自愈：worker 崩溃自动重启
- 安全隔离：root 绑端口后 worker 降权

## 五、Nginx 编译部署

### 5.1 依赖检查

```
GCC:     15.2.0 ✓
Make:    4.4.1 ✓
libssl-dev:  3.5.5 ✓
zlib1g-dev:  1.3.1 ✓
PCRE-dev:    未安装（跳过 rewrite/gzip 模块）
```

### 5.2 配置

```bash
cd /home/chaste/nginx
./auto/configure \
  --prefix=/home/chaste/nginx-deploy \
  --without-http_rewrite_module \
  --without-http_gzip_module
```

配置选项说明：
- `--prefix`：安装目录
- `--without-http_rewrite_module`：因缺少 PCRE 库，禁用 rewrite 模块
- `--without-http_gzip_module`：因缺少 zlib 开发包（实际有但简化编译）

### 5.3 编译

```bash
make -j$(nproc)
```

编译产物：`objs/nginx`

### 5.4 安装

```bash
make install
```

安装目录结构：

```
/home/chaste/nginx-deploy/
├── sbin/nginx      ← 可执行文件
├── conf/
│   ├── nginx.conf  ← 主配置文件
│   ├── mime.types  ← MIME 类型映射
│   ├── fastcgi.conf
│   ├── fastcgi_params
│   ├── uwsgi_params
│   └── scgi_params
├── html/
│   ├── index.html  ← 默认欢迎页
│   └── 50x.html    ← 错误页
└── logs/
    ├── error.log   ← 错误日志
    └── access.log  ← 访问日志
```

### 5.5 配置修改

监听端口从 80 改为 8080（非 root 用户可用）：

```nginx
server {
    listen       8080;
    server_name  localhost;

    location / {
        root   html;
        index  index.html index.htm;
    }
}
```

### 5.6 启动

```bash
/home/chaste/nginx-deploy/sbin/nginx -c /home/chaste/nginx-deploy/conf/nginx.conf
```

### 5.7 验证

```bash
curl http://localhost:8080/
# 返回: Welcome to nginx! 页面
```

## 七、编译停用的模块

由于未安装对应开发库，以下模块在此次编译中未启用：

| 模块 | 缺失依赖 | 影响 |
|------|---------|------|
| http_rewrite_module | libpcre-dev | 无法使用 rewrite/if/set 指令 |
| http_gzip_module | zlib 开发头 | 无法压缩响应内容 |
| http_ssl_module | （可选） | 无 HTTPS 支持 |

如需启用，安装依赖后重新编译：

```bash
sudo apt install libpcre2-dev
./auto/configure --prefix=/home/chaste/nginx-deploy  # 不加 --without
make && make install
```

## 八、Nginx 能做什么

| 用途 | 说明 |
|------|------|
| 静态文件服务 | 托管 HTML/CSS/JS/图片等静态资源 |
| 反向代理 | 将请求转发到后端应用（Node.js/Java/Python） |
| 负载均衡 | 将流量分发到多个后端实例 |
| SSL/TLS 终止 | 处理 HTTPS 加密和解密 |
| HTTP 缓存 | 缓存后端响应，加速访问 |
| 流媒体 | MP4/FLV 流式传输 |
| TCP/UDP 代理 | 四层负载均衡（stream 模块） |
| 邮件代理 | POP3/IMAP/SMTP 代理 |
