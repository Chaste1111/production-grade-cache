## 1. OSI 七层模型 vs TCP/IP 四层模型（本项目涉及的位置）

```
┌─────────────────────────────────────────────────────────────────────┐
│                         OSI 七层模型                                 │
├──────┬──────────────────────────────────────────────────────────────┤
│  7   │  应用层    ←── HTTP、HTTPS、DNS（代理在这里工作）             │
│      │               请求解析、URL解析、缓存判断、头修改              │
├──────┼──────────────────────────────────────────────────────────────┤
│  6   │  表示层    ←── TLS/SSL 加密（CONNECT 隧道中浏览器和目标       │
│      │               之间才涉及，代理不参与、看不懂加密内容）          │
├──────┼──────────────────────────────────────────────────────────────┤
│  5   │  会话层    ←── （HTTP 无状态，不涉及）                         │
├──────┼──────────────────────────────────────────────────────────────┤
│  4   │  传输层    ←── TCP（三次握手、四次挥手、流量控制、拥塞控制）    │
│      │               socket() / connect() / listen() / accept()      │
├──────┼──────────────────────────────────────────────────────────────┤
│  3   │  网络层    ←── IP（路由选择、IP地址、子网划分）                │
│      │               gethostbyname() DNS解析                         │
├──────┼──────────────────────────────────────────────────────────────┤
│  2   │  数据链路层 ←── 以太网帧、MAC地址、ARP                         │
├──────┼──────────────────────────────────────────────────────────────┤
│  1   │  物理层    ←── 光纤、双绞线、无线电波                          │
└──────┴──────────────────────────────────────────────────────────────┘

本项目代码主要工作在 应用层（HTTP协议） 和 传输层（TCP socket编程）
```

## 3. TCP 三次握手 + 四次挥手（传输层核心）

```
┌─────────────────── 三次握手（建立连接）────────────────────┐
│                                                           │
│  客户端                      服务器                        │
│  CLOSED                     LISTEN                        │
│    │                          │                           │
│    │── SYN, seq=x ──────────→│  SYN_SENT                  │
│    │                          │  → SYN_RCVD               │
│    │←── SYN+ACK, seq=y,      │                           │
│    │         ack=x+1 ────────│                           │
│    │  ESTABLISHED             │                           │
│    │── ACK, seq=x+1,         │                           │
│    │        ack=y+1 ────────→│  ESTABLISHED              │
│    │                          │                           │
│    每次握手的数据包：                                         │
│    序列号(seq): 标记数据包编号                                │
│    确认号(ack): 告诉对方"收到了，下一个发这个号"               │
│    标志位: SYN(建连) ACK(确认) FIN(断开) RST(重置)           │
└───────────────────────────────────────────────────────────┘

┌─────────────────── 四次挥手（断开连接）────────────────────┐
│                                                           │
│  主动关闭方                   被动关闭方                      │
│  FIN_WAIT_1                                                │
│    │── FIN, seq=u ──────────→│  CLOSE_WAIT                │
│    │                          │                           │
│  FIN_WAIT_2                  │  (可能还有数据要发)          │
│    │←── ACK, ack=u+1 ───────│                           │
│    │                          │                           │
│  TIME_WAIT                   │  LAST_ACK                  │
│    │←── FIN, seq=v ──────────│                           │
│    │                          │                           │
│    │── ACK, ack=v+1 ────────→│  CLOSED                    │
│    │                          │                           │
│  TIME_WAIT (等 2MSL ≈ 4分钟)                                │
│  CLOSED                                                     │
│                                                           │
│  TIME_WAIT 的作用：                                          │
│  - 确保最后的 ACK 对方收到了                                  │
│  - 防止旧连接的数据包干扰新连接                                │
└───────────────────────────────────────────────────────────┘
```

## 4. HTTP 协议报文结构（应用层核心）

```
┌───────────────── HTTP 请求报文 ─────────────────┐
│                                                 │
│  请求行：  GET http://www.baidu.com/ HTTP/1.1   │
│            ↑   ↑                        ↑       │
│          方法  URL(正代特有，完整地址)   协议版本  │
│                                                 │
│  请求头：  Host: www.baidu.com                  │
│           User-Agent: curl/8.18.0               │
│           Accept: */*                           │
│           Proxy-Connection: Keep-Alive           │
│                                                 │
│  空行：    \r\n                                 │  ← 头体分隔
│                                                 │
│  请求体：  (GET 请求通常为空)                    │
└─────────────────────────────────────────────────┘

┌───────────────── HTTP 响应报文 ─────────────────┐
│                                                 │
│  状态行：  HTTP/1.1 200 OK                      │
│            ↑       ↑   ↑                        │
│          协议版本  状态码 原因短语                 │
│                                                 │
│  响应头：  Content-Type: text/html               │
│           Content-Length: 30522                 │
│           Connection: close                     │
│                                                 │
│  空行：    \r\n                                 │
│                                                 │
│  响应体：  <!DOCTYPE html>                      │
│           <html>...</html>                      │
└─────────────────────────────────────────────────┘

常见状态码：
  200 OK           成功
  301 Moved        永久重定向
  302 Found        临时重定向
  403 Forbidden    禁止访问（黑名单返回这个）
  404 Not Found    资源不存在
  502 Bad Gateway  网关错误（target连不上）
```

## 5. 正向代理 vs 反向代理（网络拓扑区别）

```
┌──────────── 正向代理（本项目）──────────────────────────────┐
│                                                            │
│  浏览器 ────→ 代理(8888) ────→ 百度                        │
│   │              │               │                          │
│   │ 知道代理      │ 解析URL       │ 不知道有代理              │
│   │ 主动配了      │ 自己连目标    │ 以为请求来自代理          │
│   │ -x 8888      │ 改造请求      │                          │
│                                                            │
│  请求: GET http://www.baidu.com/ HTTP/1.1  ← 完整URL       │
│  转发: GET / HTTP/1.1                      ← 路径          │
└────────────────────────────────────────────────────────────┘

┌──────────── 反向代理（Nginx）──────────────────────────────┐
│                                                            │
│  浏览器 ────→ Nginx:80 ────→ Tomcat:8080 ────→ MySQL       │
│   │              │                │              │          │

│   │ 就是目标      │ 负载均衡       │              │          │
│                                                            │
│  请求: GET /api/data HTTP/1.1            ← 只写路径         │
│  Host: nginx.com                                          │
```

## 7. epoll 事件驱动（内核到用户态的数据流）

```
┌─────────────── 网卡到 epoll 的完整数据路径 ──────────────────────┐
│                                                                 │
│  ① 网卡收到电信号                                                  │
│     └→ 转成二进制                                                  │
│     └→ DMA 写入内存 ring buffer                                    │
│     └→ 触发硬件中断                                                │
│                                                                 │
│  ② 内核中断处理（下半部）                                           │
│     └→ 网卡驱动: 从 ring buffer 取数据                              │
│     └→ 以太网层: 拆 MAC 头，找到 IP                                  │
│     └→ IP 层: 拆 IP 头，找到 TCP                                   │
│     └→ TCP 层: 拆 TCP 头，根据端口找到 socket                       │
│     └→ 数据写入 socket 的接收缓冲区 (sk_buff)                       │
│                                                                 │
│  ③ epoll 通知                                                     │
│     └→ 检查: 这个 socket 注册到 epoll 了吗?                         │
│         ├→ 注册了 → 加入就绪链表                                    │
│         └→ 有进程在 epoll_wait 睡觉?                                │
│             └→ 唤醒进程                                            │
│                                                                 │
│  ④ 用户态代理程序                                                  │
│     └→ epoll_wait 返回                                            │
│     └→ events[0].data.ptr → Connection*                           │
│     └→ 根据 conn->state 推进一步                                   │
│     └→ 回到 epoll_wait 继续睡                                     │
│                                                                 │
│  ┌───────────────────────────────────────────────────────┐      │
│  │  epoll 内部结构 (内核空间)                              │      │
│  │                                                       │      │
│  │  ┌─────────────────────┐    ┌──────────────────┐     │      │
│  │  │  红黑树 (注册的全部fd) │    │  就绪链表(有数据的)  │     │      │
│  │  │                     │    │                  │     │      │
│  │  │  listen_fd=4        │    │  browser_fd=7 ─┐ │     │      │
│  │  │  browser_fd=7 ──────┼────┼──────────────→│─┼──→ │      │
│  │  │  target_fd=8        │    │  target_fd=8 ──┘ │     │      │
│  │  │  browser_fd=9       │    └──────────────────┘     │      │
│  │  └─────────────────────┘                              │      │
│  │                                                       │      │
│  │  查找: O(log n)  删除: O(1)  通知: O(1)              │      │
│  └───────────────────────────────────────────────────────┘      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 9. 缓存数据结构（LRU + TTL）和 TCP 无关，纯粹是应用层优化

```
┌─────────────── 链表 (双向) ────────────────────┐
│                                                │
│  头部 (刚访问/新插入)          尾部 (最久没用)     │
│  ┌─────────────────┐          ┌──────────────┐ │
│  │ baidu.com       │→        │ google.com    │ │
│  │ expire=17:05    │          │ expire=17:02  │ │
│  │ "<html>...</html>"│       │ "<html>...</html>"│
│  └─────────────────┘          └──────────────┘ │
│                                                │
│  淘汰方向 → 尾部淘汰                             │
│  移动方向 → 访问就移到头部                        │
└────────────────────────────────────────────────┘
          ↑                       ↑
          │ 哈希表 O(1) 定位       │
          │                       │
     ┌────┴───────────────────────┴───┐
     │ unordered_map<string, iterator> │
     │ "baidu.com" → 指向头部          │
     │ "google.com" → 指向尾部         │
     └────────────────────────────────┘
     
     TTL 检查: get()时先看 expire_time 过了没有
              过了 → 删掉，返回未命中
```
